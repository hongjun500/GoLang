# 方法和接口

## 方法

- 一类带特殊的 **接收者（receiver）** 参数的函数，接收者写在 `func` 关键字和方法名之间
  > 这里的接收者可以是结构体类型，还包括基本数据类型、切片、映射等
- 方法是一个特殊类型的函数，它绑定到一个类型上，并且只能通过该类型的实例来调用。方法是在结构体类型上定义的，因此它们可以访问该类型的字段
- ~~特殊接收者不能是int或者string等内建类型（基本数据类型）~~
- 方法本质就是"第一个参数是接收者的函数"：`t.Info()` 在语义上等价于 `Teacher.Info(t)`

```go
func (t Teacher) Info()  { /* 值接收者 */ }
func (s *Student) Info() { /* 指针接收者 */ }
```

### 可以在哪些类型上定义方法

- 可以在本包内定义的命名类型上定义方法，包括 type MyInt int 这种以基本类型为底层类型的自定义类型
- 不能直接给内建类型（int、string…）或其它包的类型定义方法 —— 必须先在本包定义一个新的命名类型
- 方法必须与其接收者类型定义在同一个包内（Go 硬性规则，无法绕过）

### 值接收者 vs 指针接收者

1. 值接收者：方法拿到的是接收者的**一份副本**

- 对字段"重新赋值"（含切片头、map 头）只改副本，**外部不可见**
- 但若字段是引用类型（`slice/map`/`指针`），通过副本去写元素/解引用修改，改的是同一块底层数据，外部可见！（本章 `Teacher.WorkAddress` 就是例子，高频误区）

2. 指针接收者：方法拿到指向原对象的指针，所有修改直接作用于原对象，外部全部可见

### 调用便利 vs 方法集(method set)——最易混淆的点

1. **调用层面**：只要变量可寻址，`Go` 会自动取地址/解引用，值/指针都能 `.Method()`
   - 反例：map 元素、函数返回的临时值不可寻址，不能调用其指针接收者方法

2. 接口实现层面（method set，决定能否赋给接口）
   - `T` 的方法集：只含值接收者方法。
   - `*T` 的方法集：含值接收者 + 指针接收者方法。
   - 结论：指针接收者方法只属于 `*T`；若接口方法用指针接收者实现，只有 `*T` 能赋给接口，`T` 值不行
   > **接收者为指针类型的方法被调用时，接收者既能为值又能为指针**

| 方法定义                 | T 能调用       | *T 能调用 | T 满足接口 | *T 满足接口 |
| ------------------------ | -------------- | --------- | ---------- | ----------- |
| `func (t T)` 值接收者    | ✅             | ✅        | ✅         | ✅          |
| `func (t *T)` 指针接收者 | ✅（可寻址时） | ✅        | ❌         | ✅          |

> 本章 `student` 必须存 `*Student`：因为 `Info()` 用了指针接收者，`Student` 值不满足 `Study` 接口。

### 选型准则

- 需要修改接收者、结构体较大、含 `sync.Mutex` 等不可复制字段 → 用指针接收者。
- 小而不可变的类型（坐标、时间）→ 可用值接收者。
- 同一类型的所有方法接收者尽量保持一致（要么全值、要么全指针），避免方法集混乱

## 接口

```go
package main

import (
	"bytes"
)

// ReadWrite 普通接口
type ReadWrite interface {
	Read(b bytes.Buffer) bool
	Write(b bytes.Buffer) bool
}

type Lock interface {
	Lock()
	Unlock()
}

// File 接口嵌套接口
type File interface {
	ReadWrite
	Lock
	Close()
}

```

- 接口定义一组方法签名；**任何实现了这组方法的类型都自动、隐式地满足该接口（无需 `Java` 式的`implements`）
- 接口是一种类型，它规定了变量有哪些方法
- 接口类型的变量不能调用实现了接口类型定义的方法之外的方法
- 接口类型可以作为函数的参数或返回值
- 接口类型可以用于判断一个变量是否实现了某个接口
- 接口可嵌入其它接口进行组合。
- 空接口 `interface{}`（`Go` 1.18 起有等价别名 `any`）不含任何方法，可存放任意值

### 类型断言与 `type switch`

```go
// comma-ok：安全断言，失败不 panic
if t, ok := s.(Teacher); ok { /* 使用 t */ }

// type switch：一次区分多种类型
switch v := s.(type) {
case Teacher:   // v 是 Teacher
case *Student:  // v 是 *Student
default:        // 其它
}
```

- `x.(T)` 单返回值形式断言失败会 `panic`；`v, ok := x.(T)` 形式失败只是 `ok=false`，更安全

### 编译期实现断言

```go
var _ Study = Teacher{}       // 保证 Teacher 实现了 Study
var _ Study = (*Student)(nil) // 保证 *Student 实现了 Study
```

> 一行零开销的"契约检查"，方法签名写错会在编译期立即暴露

## 底层原理解析

- **接口的内部结构**
  - 非空接口 `iface：{ itab, data }`。`itab` 存"接口类型 + 具体类型 + 方法地址表"，`data` 指向具体值
  - 空接口 `eface：{ _type, data }`，只存类型指针与数据指针
- **接口调用是动态派发**：经 `itab` 里的函数指针间接调用，比直接调用多一层跳转，且常阻止内联，热点路径需留意
- **接口装箱会逃逸**：把具体值赋给接口，值通常逃逸到堆并复制一份，可用 `go build -gcflags="-m"` 观察

## 避坑指南
- 含 `nil` 指针的接口 `≠ nil` 接口本质是 (类型, 值) 二元组，只有类型和值都为 `nil` 时接口才 `== nil`
```go
var p *Student = nil
var i Study = p
fmt.Println(p == nil) // true
fmt.Println(i == nil) // false！动态类型是 *Student，非 nil
```
> 最典型的翻车是 `error`：函数返回"值为 `nil` 但类型非 `nil`"的 `error` 变量，调用方 `if err != nil` 恒为真。返回成功时应直接 `return nil`，别 `return` 一个 `nil` 的具体错误指针

- 指针接收者方法不能在不可寻址的值上调用：如 m[k].Method()（指针接收者）无法编译，需先取到变量
- 值接收者 + 引用类型字段：误以为值接收者"完全不影响外部"，却通过 `map/slice` 元素写把外部数据改了（本章 `Teacher.WorkAddress`）
- 接口比较可能 `panic`：接口可用 `==`，但底层动态类型是不可比较类型（`slice/map/func`）时，比较会运行时 `panic`
- 优先小接口：`Go` 提倡小接口（如 `io.Reader` 只有一个方法），接口应由使用方按需定义，而非实现方一次塞满

> 小写字母开头的函数只能在本包内使用，大写字母开头的函数才可以在其它包中使用，此规则同时适用于变量和类型
