# 数组、切片和 `map`

## 数组

- 数组的声明方式是 `var name [len]T` 如果 `len` 是三个点(...)那么**数组长度将由初始化元素个数在编译期推断**，结果仍是定长数组
- 元素有零值，根据数组类型 `T` 决定，（`int`→`0`，`string`→`""`）
- 长度是类型的一部分，因此不可变；`[2]int` 与 `[3]int` 是不同类型。_**数组的长度是常量，因此是不可变的**_
- **数组无法比较大小，只能比较是否相等可用 `==` / `!=` 判断相等，长度和值一样才相等**

## 切片

- 切片的底层就是指向数组的指针，在引用某个切片 `A` 得到另一个切片 `B` 时(切片截取)，改变切片 `B` 的元素内容，切片 `A`
  的内容也会随之改变(这**本质上就是指针的特征**)

> 切片头包含：指向底层数组的指针、长度 `len`、容量 `cap`

- 切片的声明方式是 `var varName []T`，其中 `T` 是切片中元素的类型
- **nil 切片** 可以 `len` / `cap` / `range` / `append`；对 nil 切片**按下标赋值**会 panic

> _**切片必须要初始化才能使用**_

- 截取得到的新切片通常与原切片**共享底层数组**，改其中一个可能影响另一个

```go
// 由于数组是定长的(数组的长度是其类型的一部分，因此数组不能改变大小)
// 因此在Go中更多使用的是切片，(切片是长度可以变化的数组)切片和数组又息息相关
var arrSlice []string
if arrSlice == nil {
fmt.Println("切片arrSlice是 nil")
}
// arrSlice[0] = "hello" // 这一行编译通过，但是运行报错，切片只是声明了但没有初始化
// 本质上切片的底层实现是一个指向数组的指针，在没有存入一个具体的数组之前，就是 nil
// arrSlice = []string{""}  直接是以这种简单方式就可以使用
// 或者通过内置函数 make 进行初始化 make([]T, len, cap)
```

### 切片的创建方式有两种

    1. 基于数组/切片截取创建
       ~~~go
       // 创建一个数组
       var arr = [5]int{1, 2, 3, 4, 5}
       // 创建一个切片
       var slice = arr[1:3]
       ~~~

    2. 通过内置函数 `make` 创建，`make([]T, len, cap)`，其中 `len` 是切片的长度，`cap` 是切片的容量
       ~~~go
       // 创建一个切片
       var slice = make([]int, 5, 10) // len=5, cap=10
       ~~~
       > `len` 必须指定，`cap` 可以不指定并且在指定 `cap` 的值时 `cap` 的值必须要大于等于 `len`

### 长度与容量

- `len` 切片中可见元素个数
- `cap` 从切片起始位置到底层数组末尾还可容纳的元素个数
- 切片的长度和容量可以通过内置函数 `len` 和 `cap` 来获取，切片的长度和容量可以通过 `append` 函数来改变
- `append` 在容量不足时会分配新底层数组（是否扩容、扩多少随实现/版本变化）

### 截取写法

> 半开区间，包含 begin，排除 end：

- 切片的截取写法
  - `slice[begin:end]` 半开区间，包含 begin，排除 end
  - `slice[:end]` 从下标 0 开始到下标 end-1 的元素
  - `slice[begin:]` 从下标 begin 开始所有的元素
  - `slice[:]` 从下标 0 开始到下标最大值位置的元素(即所有元素)
  - `slice[low:high:max]`：额外限制 cap = max-low

> 切片的截取时最终得到的切片会根据 `slice[begin:end]`中上界下界的值进行界定，而写法有多种要么没定义上界要么没定义下界，总的来说*下界定义了
> 就是最后一个元素取下标为下界值减一，反之则是都要*(加深自我理解)

### 切片的遍历

1. 通过 `for` 循环加上 `range` 关键字

```go
for index, value := range slice
```

2. 通过 `for` 循环加上 `len` 函数和下标

```go
    for i := 0; i < len(slice); i++
```

### 切片的拷贝与追加

1. `copy(dst, src)`: 按两者较短长度拷贝,返回拷贝个数

> copy 只是把 src 的元素搬运到 dst 已有的底层数组里,本身不分配内存。dst 是否和 src 共享底层数组,完全取决于 dst 是怎么来的——如果 dst 是提前 make 出来的独立切片,拷贝后自然不共享;但 copy 同样支持同一底层数组内部的重叠拷贝(类似 memmove),常用于原地删除元素:

```go
s := []int{1, 2, 3, 4, 5}
copy(s[0:], s[1:]) // s 变成 [2 3 4 5 5]
s = s[:len(s)-1]   // [2 3 4 5],实现"删除首元素"
```

还有一个常用重载:直接把字符串拷进 []byte,省去类型转换:

```go
b := make([]byte, 5)
copy(b, "hello")
```

2. `append` 是否共享底层数组,取决于容量够不够

   ```go
   s := make([]int, 3, 5)   // len=3, cap=5
   s2 := append(s, 100)     // cap 够(5>3),直接在原数组扩展 → s2 和 s 共享底层数组!
   s2[0] = 999              // s[0] 也会跟着变
   ```
   - 容量够 → 在原底层数组上扩展,新旧切片共享内存,修改一个可能"意外"影响另一个(典型 bug:函数内部对传入的 slice append 后,调用方那边的数据被改了)
   - 容量不够 → 才会分配新数组,此时两者才真正独立

3. 想要"真正复制一份"、强制走新数组,有两种等价写法

```go
    dup := append([]T(nil), src...)

    // 或者,一次性分配到位、通常更推荐:
    dup := make([]T, len(src))
    copy(dup, src)
```
> 前者本质是"从 nil 开始 append",一定会分配新数组,但底层按扩容策略增长,src 很大时可能多几次分配;后者一次 make 分配到位,更高效。二者选一即可,写法上后者更常见。
4. append(s, elems...):追加元素;elems 也可以是同类型切片(需 ... 展开)
```go
b := append([]byte("Hello, "), "World"...)
```

## map

> 元素对的无序集合

- `map` 是引用类型，当将 `map` 传递给函数时，函数内部对 `map` 的修改会影响到原来的 `map`
- `map` 的声明方式 `var mapName map[K]V`，其中 `K` 是 `key` 的类型，`V` 是 `value` 的类型
- `key` 的类型必须满足可比较的要求，即 `==` 和 `!=` 操作符可用于该类型，`value` 的类型没有限制

> 在 `go` 中因为线程安全的问题将 `map` 分为了无锁的 `map` 和自带锁的 `sync.Map`

### 无锁 `map`

- 声明方式 `var mapName map[K]V`
- 初始化使用内置函数 `make(map[K]V)`
- `map` 的取值 `mapName["key"]`,该表达式会返回两个元素，一个是值，一个是布尔类型的判断 `key` 是否存在于该 `map`中
- `map` 的遍历可以使用 `for` 循环加上 `range` 关键字
- `map` 删除元素可以使用内置函数 `delete(m map[Type]Type1, key Type)`,并且如果 `key` 不存在也不会有问题(排除极端情况)

### 有锁（并发场景） `sync.Map`

```go
// 声明方式
var syncMap sync.Map
// 初始化方式
syncMap = sync.Map{}
// 添加元素
syncMap.Store("key", "value")
// 获取元素
syncMap.Load("key")
// 删除元素
syncMap.Delete("key")
// 遍历元素
syncMap.Range(func (key, value interface{}) bool {
fmt.Println(key, value)
return true // 返回 false 结束遍历
})
```

> 此种类型操作都是基于自带函数

### map 与切片组合
- `[]map[K]V`：切片元素是 `map` 时，每个元素都要单独 `make`，否则是 `nil`
- `map[K][]V`：值为切片时，可直接 `append` 到 `m[k]`（注意接收返回值）
