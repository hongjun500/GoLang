# 函数

~~在 `go` 中所有东西都是变量, 函数也是变量的一种，因此函数名的命名规则也和变量一样~~

- **函数是 Go 的"一等公民"（first-class citizen）**：可以赋值给变量、作为参数或返回值、存入数据结构。函数名的命名规则与标识符一致。
- 使用关键字 `func` 来声明和定义

```go
func funcName (parameter-list) (result-list) {
body
}
```

- 函数的主要特点
  - 函数可以使用 `...T` 的方式来定义参数，表示参数的个数是可变的
  - 函数可以返回多个值
  - 函数的返回值可以被命名，并且像变量那样使用，即
  > ~~返回值可以用括号显示声明出来，即一开始就定义好，那么在函数体中可以直接调用，函数最终直接一个 `return` 返回~~
  > 命名返回值一开始就在签名里声明好，函数体内可直接赋值，最后用一个裸 return 返回
  - 函数可以作为变量的值
  - 函数可以作为参数和返回值
  - ~~隐藏函数名的匿名函数可以是闭包的，闭包是一个函数值，_它引用了其函数体之外的变量或者常量的内存地址，而不是它们的值_~~
  - 匿名函数可以形成闭包：闭包捕获的是其函数体之外的变量本身（共享同一存储单元），而不是变量此刻的值快照

## 参数传递：Go 只有值传递

> `Go` 语言只有值传递（pass-by-value），不存在 `C++` 那样的"引用传递"

- ~~函数的参数可以是值传递，也可以是引用传递，即~~
  - ~~值传递：传递的是值的副本，不会改变原有的值~~
  - ~~引用传递：传递的是值的内存地址，会改变原有的值~~
- 一切参数都是拷贝：调用时把实参复制一份给形参
- 之所以"传切片/`map`/指针能改到外部数据"，是因为这份被拷贝的值本身就含有指向底层数据的指针（切片头里的指针、`map` 的 `hmap` 指针、指针变量存的地址），函数内外操作的是同一块底层数据 —— 但切片头、指针变量本身仍然是副本
- 反证：函数内 `s = append(s, x)` 触发扩容后，调用方切片的 `len` 不会变化，正说明切片头是副本。

| 传入类型               | 拷贝了什么                      | 能否改到外部原数据                  |
| ---------------------- | ------------------------------- | ----------------------------------- |
| `int/string/struct` 等 | 整个值                          | 否                                  |
| 指针 *T                | 地址（副本）                    | 是（通过解引用）                    |
| `slice`                | 切片头（指针/`len/cap` 的副本） | 元素能改；`append` 扩容后不影响外部 |
| `map` / `chan`         | 内部指针（副本）                | 是                                  |

## 匿名函数与闭包

- 匿名函数可以定义在函数内部、作为函数的参数或者返回值
- ~~匿名函数的闭包可以在函数内部引用外部变量或者外部常量，这里的**引用**其实是*引用了其内存地址，而不是引用了其值*。
  这通常不需要通过参数传递，因为匿名函数可以直接使用外部变量或者外部常量~~
- 闭包捕获的是变量本身，因此被捕获的局部变量会逃逸（escape）到堆，生命周期被延长；对该变量的后续修改，在闭包里也能看到。常量没有地址，捕获常量用的是编译期内联的值

  ```go
  func demo() {
      // 赋值给变量后调用
      fc := func(name string) int {
          fmt.Printf("匿名函数接收到的参数是：%s\n", name)
          return len(name)
      }
      fc("hello")

      // 立即执行的匿名函数 (IIFE)
      func(name string) {
          fmt.Printf("直接调用：%s\n", name)
      }("world") // 结尾的 () 表示立即调用

      // 闭包：counter 每次 +1，说明捕获的是 count 变量本身
      counter := func() func() int {
          count := 0
          return func() int { count++; return count }
      }()
      fmt.Println(counter(), counter(), counter()) // 1 2 3
  }
  ```

### `defer` 深度解析

> `defer` 会把函数调用压栈，在外层函数返回前执行。三个必须掌握的规则

1. 执行顺序：后进先出（LIFO）

```go
for i := 1; i <= 3; i++ {
    defer fmt.Println(i) // 输出 3, 2, 1
}
```

2. 参数立即求值，闭包延迟读取

```go
	word := "A"
	defer fmt.Println(word)          // 立即求值：打印 "A"
	defer func() { fmt.Println(word) }() // 闭包：打印 return 时刻的最新值 "B"
	word = "B"
```

3. defer 可以修改命名返回值

```go
func f() (result int) {
    defer func() { result *= 10 }()
    result = 1
    return // 先置 1，再被 defer 改成 10，最终返回 10
}
```

### panic 与 recover

- `recover` 只有在 `defer` 函数中调用才生效，用于把运行时 `panic` 转为可控错误，避免进程崩溃
- 惯用法：库/服务的边界处兜底，业务错误优先用 `error` 返回，而非 `panic`

```go
func safe() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("recovered:", r)
        }
    }()
    panic("boom")
}
```

## 可变参数（variadic）

- `func f(nums ...int)` 中，`nums` 本质是 `[]int`；不传参时它是 `nil`（可安全 `range`）
- 传切片用 `f(s...)` 展开；此时形参与调用方切片共享底层数组，元素写操作会外泄

## 必坑指南

- 闭包 + 循环变量
  - `Go` 1.22 之前：`for` 的循环变量整个循环共享一个，配合 `goroutine` /闭包会全部拿到同一个（通常是最后一个）值。老代码常见修复：`i := i` 影子变量，或用参数传入
  - `Go` 1.22 起：循环变量改为每次迭代一个新实例，这个坑默认消失。写代码/读老代码务必确认 `Go` 版本
- `for` 循环里滥用 `defer`：`defer` 到外层函数返回才执行，循环内打开大量文件/连接却 `defer Close()`，会导致资源迟迟不释放。应把逻辑抽成独立函数，或显式 `Close`
- 变参 + `append` 共享底层数组：对变参切片 `append` 或改元素，可能"意外"改到调用方数据；需要隔离时先 `copy` 一份
- 函数值不可比较：函数之间不能用 `==`（只能和 `nil` 比），也不能作 `map` 的 `key`。判断"是否已设置回调"用 `if fn != nil`
- 命名返回值别滥用：短函数用命名返回值 + 裸 `return` 很清爽，但长函数里裸 `return` 会降低可读性，且容易和 `defer` 修改返回值互相"打架"

## 底层原理解析

- 逃逸分析（escape analysis）：闭包捕获的变量、被返回地址的局部变量会从栈逃逸到堆。用 `go build -gcflags="-m"` . 可以观察逃逸决策
- `defer` 性能：`Go` 1.14 引入 `open-coded defer`，绝大多数场景下 `defer` 已接近零成本，不必为性能而刻意回避 `defer`；仅在极热点、每秒百万次级别的路径才需评估
- `Go` 不做尾递归优化（TCO）：深递归可能爆栈（`goroutine` 栈虽可增长但有上限），热点/深度不确定的场景优先改写为迭代
- 多返回值的实现：`Go` 通过寄存器/栈传递多返回值（ABI 相关），`(T, error)` 双返回是标准错误处理范式，几乎无额外装箱开销
